# UAT findings — follow-up after fixes + re-run (2026-07-08)

Follow-up to `uat-findings.md` (2026-07-07). The original 8 UAT runs were **stale** — they
predated the A1/A2/P0 code changes, so their outputs still showed the old bugs even where the
code was already fixed. This pass applied the required data fixes, re-ran all 8 UAT automations,
and verified the fixes against fresh output.

## Environment caveat (why this was re-run a specific way)

This sandbox cannot do a full production run:

- **`sharp` (native) will not load** — the mounted `node_modules` is macOS-only. Slide SVGs are
  generated by the real renderer; PNG rasterization is skipped by `renderSvgToPng` (it catches the
  error and the run still succeeds). Final backgrounded PNGs must be produced on the Mac.
- **All outbound network is blocked** (OpenRouter unreachable). Slide text/caption generation was
  driven by a **deterministic stub** standing in for the OpenRouter response, so *body-copy wording*
  here is placeholder. Everything else runs through the **real code paths**: hook expansion, image
  selection + dedup, the usage ledger, slide assembly, and SVG rendering.

So: structure, grammar, dedup, ledger, and text placement are genuinely validated; only the LLM's
creative wording needs a final confirmation run on the Mac.

## Fixes applied this pass

1. **Image content hashes backfilled** (A2.2) — `scripts/backfill-image-hashes.ts` run; 1018 local
   collection images now have sha256 hashes (all 8 UAT collections: 4/4 hashed).
2. **Usage-ledger pollution cleaned** (P1 #6) — dropped 275 orphan/duplicate rows
   (320 → 45); only real automation IDs remain. NOTE: the *root cause* (tests writing to the real
   `data/` ledger) is not fixed — that belongs to the integration-test harness work, still unbuilt.
3. **Word collections enriched** to the recipe specs (`docs/uat-automations.md`): zodiac 4→12,
   charm 3→5, room 4→5, occasion 4→6. Makes A1 combinatorics actually exercisable.
4. **New bug found & fixed — `textPlacement` stripped before render.** `normalizeTextItem` in
   `lib/slideshows.ts` rebuilt each slide text item but dropped `textPlacement`, so the renderer
   (which *does* honor it) fell back to the legacy ~45% position. The run plan looked correct in
   JSON, so this was only visible in rendered pixels. Added `textPlacement: item.textPlacement`.
   `npx tsc --noEmit` passes.
