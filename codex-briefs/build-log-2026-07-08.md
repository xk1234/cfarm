prefix (`curtain`→`curtains`), limit, and field-weight ranking all pass.

### B5.1 — Swipe winner score (`lib/swipe-scoring.ts` + `.test.ts`)
Deterministic 0–100 blend: engagement (log-scaled), performance ranks (`ctr_rank`/`cvr_rank`,
parsing both ordinal and "Top X%"), longevity (observed run length + `times_seen`), recency
(30-day half-life). Returns a per-component breakdown for UI transparency. Verified: rank parsing,
bounded score, long-runner beats fresh with equal engagement, winner ranks first.

### A8.1 — Auto-publish wiring (`lib/publishing.ts`, runner, route)
The roadmap's biggest gap ("cron never publishes"). Extracted the shared publish logic into
`lib/publishing.ts` (`publishPost`, `publishAutomationRun`) with an **injectable request** so the
cron path is testable offline. Wired into `createAutomationRun`: when `auto_post` is on and there's

then reflects (queued → published/failed). Refactored `POST /api/postfast/posts` to call the same
`publishPost` (single source of truth). Publishing can never fail a run.
Verified end-to-end with a stubbed PostFast: a run with an integration + `auto_post` now yields
`socialStatuses: ["published"]` and a persisted post record; failures record `failed` without throwing.
**Follow-up (needs the Mac + network):** binary media upload (rendered slides → PostFast media keys
via the `/api/postfast/upload` path). Content posts from caption + hashtags today; `media` is already
threaded so wiring upload is a small addition.

### Baseline (network on)
`npx tsc --noEmit` clean. `npx vitest run`: **325 pass / 8 fail / 1 skip**. 7 of the 8 failures are
the **`sharp` arm64 gap** (renderer emits `.svg` placeholders instead of `.png`, cascading into the

### OpenRouter — client GOOD, default model has a real defect (A3/A7/B3)
`generateSlideshowText` exercised live end-to-end: auth, strict `json_schema` structured request,

**FINDING:** the configured default text model **`google/gemini-3.1-flash-lite` reproducibly returns
`hashtags: ""`** (3/3 runs) in the *full* pipeline schema (the one with the nested `text` placeholder
object). With a trimmed metadata-only schema flash-lite fills hashtags fine, and `google/gemini-2.5-flash`
and `openai/gpt-4o-mini` both fill hashtags on the full schema. So it's a model×schema-complexity
interaction, not a client bug. **Action needed:** either bump the `slideshowText` default model or
harden the schema/prompt (e.g. reorder `hashtags` after `text`, or restate the hashtag rule last).

## Batch 2b — Grid Girl / Horizontal Girl generation vs example outputs

Converted both public templates (`template-reelfarm-6178` Grid Girl, `template-reelfarm-4860`
Horizontal Girl) via `automationTemplateToTempSlideTestingAutomation`, generated slideshow text live
(gemini-2.5-flash) across 4 passes, and compared against `data/automation-templates/example-runs.json`
(6 example runs each). Test: `lib/__live__/girl-templates.live.test.ts` (RUN_LIVE-gated). Note the
example runs store body text baked into remote reelfarm images, so body-phrasing is compared against
the template's tone directive + the example hook, and layout/collection against the example plans.

**Collections — MATCH.** Both templates wire hook→`collection-pinterest-horizontal-hooks-…`,
body→`…-horizontal-body-…`, and CTA collection resolves to the body collection, exactly as the
templates specify. (The collections themselves aren't imported into the local
`image-collections.json`, but the wiring is correct.)

**Word counts — GENERALLY MATCH, with model variance.** Content bodies land in the 6–10 spec most
passes (occasional dip to 5–6). Content titles are usually 1–2 words, but the model sometimes folds
the "the lie:/the truth:" framing into the title item and overshoots (e.g. `[4,5,6,5,4]`). Nothing in
the pipeline clamps word count — it relies on the model honoring the placeholder direction.

**Phrasing style — GENERALLY MATCH, one real deviation.** First-person and no-discourse-marker rules
are honored every pass; the "recognizing self-deception → embracing truth" progression comes through
strongly (Horizontal literally emits `the lie:` / `the truth:` pairs). **Deviation:** Grid Girl's
metadata *title* comes back Title-Case every pass (e.g. `Uncovering Wellness Lies`,
`6 Lies I Stopped Telling Myself`) despite the tone's explicit "all lowercase" rule — the body text is
lowercase but the metadata title isn't, and `normalizeTempSlideStructuredOutput` doesn't lowercase.
Horizontal Girl stays fully lowercase. Also **hashtags are frequently EMPTY here even on
gemini-2.5-flash** — so the batch-2 hashtag issue is broader than flash-lite; it's a schema-interaction
reliability problem, worth hardening regardless of default model.

**Layout — MISMATCH (the main finding).** Every generation yields **6 slides (hook + 5 content)**; the
examples have **7 (hook + 5 content + CTA)**. Root cause: there are two converters with different CTA
logic. `automationTemplateToTempSlideTestingAutomation` (the template path these use) gates the CTA
slide on `format.cta.enabled` only — which is `false` on both templates — while
`automationSchemaToTempSlideTestingAutomation` gates on `cta.slideCount > 0 || cta_slide.check`. Both
templates carry `image_collection_ids.cta_slide.check = true` (+ `cta_location: "last_slide"`), which is
the signal reelfarm used to render the 7th slide. **Fix:** make the template-path converter also honor
`cta_slide.check` so it matches the schema path and the example output.

## Batch 3 — fixes for CTA + phrasing (implemented & re-validated)

Investigated fixes empirically against live OpenRouter before committing. Experiment harness:
`lib/__live__/prompt-experiments.live.test.ts` (RUN_EXP-gated) — prompt/schema variants ×
{flash-lite, 2.5-flash} × 3 trials on Grid Girl, scoring hashtags / title-lowercase / all-lowercase /
titles-1-2 / bodies-6-10.

**What the experiments showed.** Schema reordering (hashtags after `text`) gave no lowercase/word-count
benefit; the prompt+schema `combo` even produced a truncated-JSON response once. Clear winner was
**prompt hardening**: explicit "fill EVERY field / never empty", "respect the exact word range", and a
conditional "everything lowercase" rule took gemini-2.5-flash from titleLower 0-1/3 and bodies 1/3 to
**3/3 across the board**, while flash-lite stayed 3/3.

**Implemented** (`lib/temp-slide-testing.ts` + `lib/slideshow-text-generation.ts`):
1. **CTA** — `automationTemplateToTempSlideTestingAutomation` now gates the CTA slide on
   `format.cta.enabled || cta_slide.check` (new `parseTemplateCtaSlideCheck`), matching the schema path.
2. **Prompt hardening** — `buildTempSlideUserPrompt` appends "Strict output rules" plus a conditional
   all-lowercase rule when the style requests it (`styleRequestsLowercase`).
3. **Deterministic lowercase backstop** — `normalizeTempSlideStructuredOutput` takes `{ lowercase }`;
   `generateSlideshowText` sets it from the automation's style/tone, guaranteeing lowercase regardless of
   model compliance.
4. **Robust hashtags** — `normalizeHashtags` accepts comma/space-separated tags and re-adds a leading `#`
   if omitted (previously any tag without `#` was silently dropped → empty output).

**Re-validation (live, 2 passes each template, gemini-2.5-flash):** both templates now report
**layout=MATCH (7 slides incl. CTA), lowercase=true, hashtags=present, titles 1-2, first-person,
no discourse markers** every pass; CTA copy renders correctly. Body word count is the only soft metric
still occasionally +1 over (11 vs 10) — model-dependent, no structural impact. The empty-hashtags issue
also disappears on the original terse fixture: flash-lite (default model) now returns hashtags where it
previously returned empty 3/3, and titles there stay Title-Case (correct — that tone doesn't request
lowercase). `tsc --noEmit` clean; offline `vitest run` unchanged at the same 8 env-only failures
(7 `sharp`, 1 `text-similarity`), no new regressions; `temp-slide-testing.test.ts` still 8/8.
