## Product lifecycle

```text
Template or automation
  -> input collections and runtime variables
  -> manual or scheduled run
  -> slideshow, video, or social draft output
  -> review and edit
  -> PostFast publication or manual-published evidence
  -> calendar and analytics projections
```

## Documentation rules

- `docs/components/` explains frontend ownership and reuse boundaries.
- `docs/routes/` explains browser and server entry points.
- `docs/collections/` is the canonical collection-type, CRUD, and usage
  reference.
- `docs/reference/` contains implementation contracts that describe shipped
  behavior.
- `docs/tabs/` contains user-facing workflow notes for product surfaces.
- `docs/roadmap/` contains planned work that is not necessarily implemented.
- Update the relevant document in the same change as a route, persistent shape,
  lifecycle rule, or major component boundary.

The repository-level `docs/README.md` remains a compact maintainer index. This
page is the rendered documentation home at `/docs`.
